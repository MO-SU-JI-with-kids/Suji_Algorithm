# 🏆 알고리즘 스터디

## 📅 스터디 규칙

- **기간:** 2026.02 ~ 종료 시까지
- **일정:** 매일 당일에 주어진 문제 풀이 완료
- **필수 과제**
  1. **[Daily] 하루 할당량 풀기 (실버 2개 or 골드 1개)**
  2. **[Weekly]** 매주 어려운 문제 1개 선정하여 수도코드 작성 및 풀이
- **문제 추가:** 이미 풀었던 문제일 경우, 백준 문제집에 본인이 원하는 문제를 자유롭게 추가할 수 있음.
- **방식:** PR 없이 각자 폴더에 소스코드 업로드 (**main 브랜치 직접 Push**)
- **벌칙:** 미제출 시 [벌금 5천원!!]

---

## 📂 저장소 구조

모든 작업은 본인 이름으로 된 폴더 내에서만 진행한다. 다른 사람의 폴더는 절대 건드리지 않는다.

```text
root/
├── README.md
├── [본인이름]/
│   ├── BJ_1000.java
│   └── BJ_2000.py
└── pseudocode/
    └── week01/
        ├── 홍길동.java
        └── 김길동.java
```

## 🚀 데일리 문제 풀이 프로세스

1.  매주 시작 전 (최신 코드 가져오기)
    작업 전, 다른 팀원이 업데이트한 내용을 반드시 먼저 받아와야 충돌을 방지할 수 있다.

        git pull origin main

2.  문제 풀이
    본인 이름 폴더 안에 해당 주차 폴더를 생성하고 코드를 작성한다.

        예: gildong/week01/BJ_1000.py

3.  코드 제출 (Push)
    문제를 다 풀었으면 아래 명령어로 서버에 올린다.

        git add '파일명'

        git commit -m "weekN BJ_문제번호"

        git push origin main

⚠️ 주의: push 할 때 에러가 발생하면, 그 사이에 다른 팀원이 코드를 올린 것이다. 당황하지 말고 다시 `git pull origin main`을 실행한 뒤에 push 해라.

## 🚀 주간 공통 문제 프로세스

매주 한 번, 가장 어려운 문제를 함께 해결하기 위해 아래 프로세스를 따른다.

1. 문제 공유 (Issue 생성)
   Issues 탭에서 공통 문제 공유를 위한 이슈를 생성합니다.

   `제목: [Week N]  문제명`

2. 로직 논의
   생성된 이슈를 보며 다 같이 모여 어떻게 풀지 의논합니다.

- 각자의 아이디어나 의사결정 과정을 서로 공유하며 수도코드를 작성합니다.
- 논의된 수도코드는 캡처하여 해당 이슈의 댓글에 업로드합니다.

3. 개별 풀이 및 제출

- 논의한 내용을 바탕으로 스터디가 끝난 뒤 각자 코드를 구현합니다.

- 구현된 파일은 `pseudocode/weekN/` 폴더에 업로드합니다.

- 커밋 메시지: `[이름] weekN 문제이름 #이슈번호`

- 이 형식을 지키면 해당 이슈와 여러분의 풀이 기록이 자동으로 연결됩니다

## 💬 커밋 컨벤션

로그를 깔끔하게 유지하기 위해 아래 형식을 지킨다.

데일리 제출: `[이름] weekN BJ_문제번호`

공통 문제 제출: `[이름] weekN 문제이름 #이슈번호`

## 💡 유의사항

개인 폴더 엄수: 본인 폴더 외의 파일이나 설정은 수정하지 않는다.

README 충돌 주의: 현황판 업데이트 시 여러 명이 동시에 수정하면 충돌이 날 수 있다. 수정 후 바로 push 하는 습관을 들이자.
